# 프로젝트 개발 및 코딩 규칙 (Developer Guidelines)

이 프로젝트(`FortuneWeather`)의 소스코드를 편집하거나 신규 기능을 추가할 때 모든 AI 코딩 에이전트와 개발자가 영구적으로 준수해야 하는 철칙입니다.

## 🚨 최우선 철칙: 하드코딩 금지 (No Hardcoding)

1. **상태 관리 및 조건문 하드코딩 금지**
   - UI의 편집 상태(`isEditing` 등)나 화면 분기 상태를 특정 UI 상태(`SajuUiState.Initial` 등)에 결합할 때 무조건적인 하드코딩 상수(`true`, `false`)를 강제 주입하지 마십시오.
   - 반드시 로컬 데이터의 존재 여부(`birthDate.isBlank()`)나 비즈니스 데이터 흐름에 연동하여 상태가 논리적이고 유동적으로 결정되도록 하십시오.

2. **상수 관리 및 설정 분리**
   - 매직 넘버, API 주소, 기본 레이아웃 수치, 한글/영어 정적 텍스트 등을 코드 내부에 직접 하드코딩하지 마십시오.
   - 설정값은 `BuildKonfig`, `local.properties`, 혹은 별도의 전용 상수 오브젝트(`WeatherConstants` 등)에서 체계적으로 관리해야 합니다.

3. **반응형 및 동적 설계 준수**
   - 기기 화면 크기, 네트워크 상태 변화, 시간 경과에 항상 유연하게 대응하도록 설계하십시오. 정적인 고정 수치를 UI 레이아웃이나 조건 분기에 하드코딩하는 행위를 일절 금지합니다.

---

*주의: 새로운 대화 세션이나 컨텍스트가 시작될 때마다, 작업에 들어가기 전 이 규칙 파일을 가장 먼저 로드하고 철저하게 준수해야 합니다.*
