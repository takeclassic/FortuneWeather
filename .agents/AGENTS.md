# 프로젝트 규칙 (Project Rules)

## 🎨 이미지 및 그래픽 에셋 생성 프로세스 규칙
- **시안 선검토 및 순차 승인 규칙:**
  앞으로 프로젝트 개발 과정에서 에셋(아이콘, 이미지 등)을 생성할 때, 개발자 임의로 여러 개를 일괄 생성하지 마십시오.
  1. 먼저 **단 1개의 디자인 대표 시안**을 생성하여 사용자에게 시각적 마크다운 형태로 제공합니다.
  2. 사용자가 해당 디자인 시안을 확인하고 **최종 승인(Ok)**을 내린 경우에만, 그와 동일한 컨셉 및 화풍을 유지하며 나머지 연관 리소스들을 생성합니다.
  3. 시안이 승인되기 전에는 **개발 코드를 변경하지 않고**, 디자인 피드백을 우선 반영하는 과정을 거칩니다.
  4. 세션이 리셋되거나 재시작되어도 이 워크플로우를 최우선 지침으로 준수해야 합니다.
